Sơ Đồ Tổng Quan

Sơ đồ tổng quan cho thấy toàn bộ hệ thống logistic Medcon được chia thành các cụm chức năng chính: booking, điều phối lấy hàng, kế hoạch chuyến, chuyến xe thực tế, giao hàng cuối, công nợ đối soát và quản trị vận hành.
Dựa trên mô hình này, mỗi chặng vận hành giữa hai điểm liền kề đều dùng chung một logic, chỉ khác vai trò tham gia, điểm đầu, điểm cuối và quyền điều phối.

Nhóm Use Case Cốt Lõi

1. Booking & Quản lý đơn hàng Tạo booking thủ công, nhận đơn từ WMS/ERP, chỉnh sửa chuẩn hóa dữ liệu, phân loại đơn, kiểm tra bắt buộc và in phiếu gửi/QR.
2. Điều phối & Lấy hàng Duyệt đơn, phân công theo tầng, ghi nhận thời gian, xác nhận người lấy, kiểm đếm và xử lý lấy hàng thất bại.
3. Phân tuyến & Kế hoạch chuyến Gom đơn theo tỉnh, tính tải, xác định tuyến, hệ thống đề xuất kế hoạch chuyến để điều phối viên duyệt.
4. Chuyến xe thực tế Xe nhận hàng, chờ xuất bến, đang chạy, bàn giao tại điểm trung chuyển và theo dõi tải/trễ/hủy/chuyển đổi tài xế.
5. Giao hàng cuối & Hoàn hàng Giao đủ, giao một phần, giao thất bại, ký biên bản, sinh QR thanh toán, xuất hóa đơn và xử lý hoàn hàng.
6. Công nợ, COD & Đối soát Theo dõi cước khách gửi, tiền thu hộ, nợ khách nhận, phí phải trả đối tác và kỳ đối soát.
7. Quản trị vận hành nền Quản lý xe, shipper, chấm công, GPS, hợp đồng điện tử, bảo dưỡng, hiệu suất và tính lương.

Sequence Diagrams

Phần dưới đây bám theo sườn sequence diagram bạn đưa, đã được làm sạch và sắp xếp lại để có thể dùng làm tài liệu phân tích hoặc chuyển thành UML sau này.
1. Booking → Điều phối → Lấy hàng Hệ thống tự validate và tính cước (internal), điều phối chỉ duyệt và phân công, tài xế là actor cuối cùng chuyển trạng thái thực thi. 
Khách gửi        Hệ thống        Điều phối        Tài xế
    |                |                |              |
    |---Tạo đơn----->|                |              |
    |                |--validate----->|              |
    |                |                |              |
    |                |--tính cước--->(internal)      |
    |                |                |              |
    |                |--Đã tiếp nhận->|              |
    |                |                |              |
    |                |<--duyệt đơn----|              |
    |                |                |              |
    |                |<--phân công----|              |
    |                |--------------->|              |
    |                |                |--giao việc-->|
    |                |                |              |
    |                |                |<--confirm----|
    |                |                |              |
    |                |<-----Đang lấy hàng------------|
    |                |                |              |
    |                |                |--lấy hàng--> |
    |                |                |              |
    |                |<----update số kiện + ảnh------|
    |                |                |              |
    |                |--Đã lấy hàng----------------->|
2. Phân tuyến → Kế hoạch chuyến → Chuyến xe Kế hoạch chuyến là đề xuất, điều phối duyệt, sau đó sinh entity mới là chuyến xe thật. 
Điều phối        Hệ thống           Tài xế / Đối tác
    |                |                     |
    |---xem đơn----->|                     |
    |                |                     |
    |---phân tuyến--->|                     |
    |                |--gom dữ liệu------->|
    |                |                     |
    |                |--đề xuất kế hoạch--|
    |<---------------|                     |
    |                |                     |
    |---chỉnh sửa--->|                     |
    |                |                     |
    |---duyệt------->|                     |
    |                |                     |
    |                |--tạo chuyến xe---->|
    |                |                     |
    |                |--gán xe/tài xế---->|
    |                |                     |
    |                |<--confirm----------|
    |                |                     |
    |                |--Xe nhận hàng------|
    |                |--Xe xuất bến-------|
    |                |--Đang chạy---------|
3. Giao hàng cuối + COD + Hoàn hàng Giao đủ, giao một phần hoặc giao thất bại đều được ghi nhận rõ. COD và hóa đơn chỉ tính trên phần đã giao. 
Tài xế        Hệ thống        Khách nhận        Kế toán
   |              |               |               |
   |---đi giao--->|               |               |
   |              |               |               |
   |---kiểm đếm-->|               |               |
   |              |               |               |
   |---ký nhận------------------->|
   |              |               |               |
   |<--xác nhận-------------------|
   |              |               |               |
   |---update kết quả------------>|
   |              |               |               |
   |              |--tính COD---->|
   |              |--tạo QR------->|

   |              |               |               |
   |              |--ghi nhận COD--------------->|
   |              |               |               |
   |              |--xuất hóa đơn--------------->|

Nhánh mở rộng:
- Giao một phần: giao 3/5 kiện -> 2 kiện hoàn, COD chỉ tính cho 3 kiện
- Giao thất bại: chọn lý do -> tạo task giao lại / hoàn hàng
4. Đối soát & Công nợ Cuối kỳ hệ thống tổng hợp phát sinh, kế toán xem báo cáo, xuất hóa đơn, đối soát COD và thanh toán.

Chi Tiết Từng Package

Package 1 - Booking & Quản lý đơn hàng
Tạo booking, nhận đơn từ kho, chuẩn hóa dữ liệu, phân loại đơn và in phiếu gửi / QR.
01
Actor chính: khách gửi hàng, phần mềm kho, điều phối viên.
Package 2 - Điều phối & Lấy hàng
Duyệt đơn, phân công theo tầng, ghi nhận lấy hàng thành công / thất bại, kiểm đếm kiện.
02
Actor chính: điều phối công ty tổng, điều phối tỉnh, hub, tài xế / shipper.
Package 3 - Phân tuyến & Kế hoạch chuyến
Gom hàng theo tỉnh, tính tải, xác định tuyến và gợi ý kế hoạch chuyến để duyệt.
03
Actor chính: hệ thống gợi ý, điều phối viên, quản lý vận hành.
Package 4 - Chuyến xe thực tế & Vận chuyển
Lập chuyến xe thật, gán xe / tài xế, xe nhận hàng, chờ xuất bến, đang chạy và bàn giao.
04
Actor chính: điều phối viên, tài xế / phụ xe, nhà xe thuê ngoài, điểm trung chuyển.
Package 5 - Giao hàng cuối, giao một phần & hoàn hàng
Giao đủ, giao một phần, thu COD, sinh QR thanh toán, xuất hóa đơn, ký biên bản và xử lý hoàn hàng.
05
Actor chính: tài xế giao hàng, khách nhận hàng, nhà cung cấp, kế toán, điều phối viên.
Package 6 - Công nợ, COD, hóa đơn & đối soát
Theo dõi cước, COD, công nợ khách gửi / khách nhận và chi phí đối tác theo kỳ.
06
Actor chính: kế toán, khách gửi, khách nhận, đối tác vận hành, quản lý.
Package 7 - Quản trị vận hành nền
Quản lý xe, nhân sự, chấm công, GPS, bảo dưỡng, hợp đồng điện tử và tính lương.
07
Actor chính: quản lý vận hành, admin, nhân sự, kế toán, đối tác.

Luồng Mô Hình Để Vẽ Sơ Đồ Sau Này

1
Booking là điểm khởi đầu của dữ liệu vận chuyển

Khách gửi tạo đơn hoặc đẩy đơn từ kho. Hệ thống logistic là nguồn sự thật cuối cùng và chuẩn hóa dữ liệu đầu vào.

2
Điều phối lấy hàng là giai đoạn chờ xác nhận chuỗi tầng

Đơn ở trạng thái Đã tiếp nhận, được phân công theo từng tầng mà không can thiệp chéo quyền.

3
Đóng hàng / kiểm đếm kiện là lớp kiểm soát bắt buộc

Ghi nhận số kiện, tình trạng kiện, ảnh hiện trạng và chênh lệch so với booking.

4
Kế hoạch chuyến là bản đề xuất để điều phối viên duyệt

Hệ thống gợi ý theo tải, tỉnh, tuyến và cut-off time. Điều phối viên chỉ kiểm tra và chốt.

5
Chuyến xe thực tế là lớp thực thi

Xe nhận hàng, chờ xuất bến, đang chạy và bàn giao tại các điểm trung chuyển hoặc điểm nhận kế tiếp.

6
Giao cuối xử lý cả giao đủ, giao một phần và hoàn hàng

Ký biên bản, ký số trên app, sinh QR thanh toán, xuất hóa đơn và ghi nhận kết quả giao.

7
Công nợ và quản trị nền là lớp quản lý dài hạn

Theo dõi cước, COD, đối tác, xe, shipper, hợp đồng điện tử, bảo dưỡng, GPS và tính lương.

Ghi Chú Thiết Kế

Bộ sơ đồ hiện tại đã đi đúng hướng “một quy trình chuẩn cho hai điểm liền kề”. Điều này rất tốt vì sau này bạn chỉ cần tái dùng cùng logic cho nhiều tầng khác nhau, thay vì viết một sơ đồ mới cho mỗi tuyến.
Gợi ý sử dụng:
- Dùng ảnh tổng quan làm slide mở đầu.
- Dùng từng package cho phần use case chi tiết.
- Dùng tài liệu markdown trước đó để viết mô tả nghiệp vụ và sequence diagram.
Nếu bạn muốn, bước tiếp theo mình có thể làm tiếp một trang HTML riêng cho sequence diagram theo từng package, hoặc mình có thể ghép thêm các file SVG/PNG thành một bộ gallery có nút chuyển qua lại giữa các package.
